# 信用扩张场景全景图
## 访问地址：https://cavno.github.io/CreditExpansion/

家庭 · 民间金融 · 公司 · 银行 · 金融市场 · 国家六个领域、五十二个典型信用扩张场景的交互式地形图。机制全球通用，案例兼及中国与世界——每个节点都附一则把该场景顶层设计推到极端一元的真实案例（温州帮、潮汕帮所代表的「民间金融」亦单列一域）。每个场景按**抵押品强度 / 杠杆 / 反身性**三轴打分，合成一个脆弱度坐标，定位它在**对冲 → 投机 → 庞氏**谱系上的位置；越往上、越红，越接近庞氏那一端。

单文件、零构建、纯前端。整个应用就是一个 `index.html`，唯一的外部依赖是思源宋体（Google Fonts 的一行 `<link>`），加载失败时自动回退到系统衬线字，不影响功能。

## 包含什么

全景图按六个领域为列，纵轴是脆弱度（背景做成绿→琥珀→赤三段明斯基带）；节点颜色表示所属明斯基带，大小表示杠杆，空心菱形是「机制 · 政策」类（如派生存款、央行工具、总量调控、担保品再抵押），与借款人的杠杆头寸区分开。其中「金融市场」一列只收录市场化／非银金融里真正创造信用的机制（回购、证券借贷、收益互换、CDS／合成 CDO、外汇掉期、担保品再抵押、券商收益凭证）；纯粹风险转移／价格杠杆型的衍生品（场内期货期权、套保利率互换、雪球与场外期权）不构成信用扩张，已刻意排除。其中「民间金融」一列收录以血缘、地缘、同乡信任替代抵押与征信的信用扩张（互保联保、标会 / 抬会、地下钱庄、民间集资、炒房团）。纵轴维度可切换为杠杆 / 反身性 / 抵押品薄弱，可按层筛选，可切到排行榜按任一维度排序，点任意节点或表行即看运作机制与真实案例——案例刻意选取把该场景顶层设计推到极端一元的样本（证券杠杆取 1929、LBO 取 RJR、TRS 取 Archegos、稳定币取 Terra、互保联保取温州 2011、主权违约取希腊等），并点明其被推向极致的结构设计。底部「读图说明 & 方法论边界」写明了打分口径与其作为代理的局限——三轴刻画的是结构脆弱度，最终的明斯基归类仍取决于现金流能否覆盖本息。

## 部署到 GitHub Pages

> 说明：部署需要你自己的 GitHub 账号与授权，我无法代为登录或创建仓库，下面两条路径任选其一，复制即可用。

### 路径 A：网页上传（最省事，无需 git）

1. 登录 GitHub，点右上角 **New repository** 新建一个仓库（例如命名 `credit-panorama`），可设为 Public。创建时不要勾选「Add a README」，保持空仓库。
2. 进入空仓库页面，点 **uploading an existing file**，把本文件夹里的 `index.html`（README 可选）拖进去，**Commit changes**。
3. 仓库页 → **Settings** → 左侧 **Pages** → Build and deployment → Source 选 **Deploy from a branch** → Branch 选 `main`、文件夹 `/ (root)` → **Save**。
4. 等一两分钟，页面顶部会出现访问地址：`https://<你的用户名>.github.io/credit-panorama/`。

### 路径 B：git 命令行

先在 GitHub 上新建一个空仓库（同上，不要带 README），然后在本文件夹内执行：

```bash
git init
git add index.html README.md
git commit -m "信用扩张场景全景图"
git branch -M main
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

若装了 GitHub CLI，前面几步可一行搞定：

```bash
gh repo create <仓库名> --public --source=. --push
```

推送完成后，仍按路径 A 的第 3 步在 **Settings → Pages** 里把 Source 指向 `main` 分支的根目录，保存即可拿到访问地址。

## 概念辨析层

方法论区新增可折叠的「概念辨析：信用 ≠ 货币 ≠ 流动性（以抵押品为铰链）」一层，含一张关系小图：货币↔信用在「贷款创造存款」处同涨，但有两个漏口——QE 造货币不造信用、社融＞M2 造信用不造货币；抵押品是连接信用与「市场/融资流动性」的铰链，上行经反身性自我强化，下行在拐点处暴力脱钩（流动性幻觉）。该图亦单独导出为 `信用-货币-流动性-关系图.svg`，可在他处复用。该层并在抽象图之下新增「六域铰链」对照：把「抵押品」这个铰链在家庭、民间金融、企业、银行、金融市场、国家六个领域分别落到实处，每行以该领域主题色串起 信用 →【抵押品（铰链）】→ 流动性，并标明铰链失灵（拐点）时的表现；末尾点明抵押品从最硬的「国债」滑到最软的「社会信任」的谱系，正是纵轴脆弱度的来源。

## 锚的迁移层

方法论区再新增可折叠的「锚的迁移：土地 → 主权信用 + 一篮子替代物」一层，含一张散点图：用与全景图同一套打分的两根轴——抵押品质量（横）× 反身性（纵）——给候选定位。土地（旧锚）在右上（质量高但反身性拉满，强大却易泡沫），沿主线迁向右下的主权信用 / 国债（新主锚，落在「稳定锚区」）；先进制造、资本市场/股权、数据资产、REITs、保障房等一篮子替代物多偏左（抵押品质量更低），黄金则标为避险 · 反杠杆（是逃离而非承载信用扩张）。该图亦单独导出为 `杠杆资产迁移图.svg`。

## 专题层：「大而不倒」案例定位

方法论区新增可折叠的「专题层：大而不倒」，含一张散点图：横轴＝系统重要性（外溢多大），纵轴＝救助结果（被放倒 → 部分救 → 债权人乃至股东全额兜底）。右上「兜底区」标注 AIG（对手方 100 美分）、花旗/美银（TARP）、两房（接管）、大陆伊利诺伊（1984，TBTF 原点）、LTCM（央行召集私人接盘）；中段「救活但有代价」标注希腊（被接管+私人减记 53%）、通用/克莱斯勒（保实体但股东清零）；左下「立威区」标注雷曼、恒大；右下「太大而救不动」标注冰岛（银行是 GDP 十倍）。判据：当放任你倒的外溢成本＞救你的成本，你才真的大而不倒——救的从来不只是你，是你背后那张对手方网络。恒大不满足该判据，故被清算而非兜底。该图亦单独导出为 `大而不倒案例定位图.svg`。案例与金额引自公开报道，为结构示意。
