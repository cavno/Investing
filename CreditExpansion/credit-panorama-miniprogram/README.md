# 信用扩张场景全景图 · 微信小程序

把网页版的「信用扩张场景全景图」改写为微信小程序，按手机与小程序的交互习惯重做：核心图表用 canvas 2d 重绘（六领域为列、纵轴脆弱度、绿—琥珀—赤三段明斯基带、按杠杆缩放的节点、空心菱形表示机制/政策），轻点任一节点从底部弹出详情卡（三轴条形、脆弱度、运作机制、把该场景顶层设计推到极端的真实案例）。另有可横滑的领域筛选、纵轴维度切换、以及按脆弱度排序的「排行榜」列表。共 52 个场景，数据与网页版一致。

## 导入与预览

1. 打开**微信开发者工具** → 新建/导入 → **导入项目**。
2. 目录选择**解压后的 `credit-panorama-miniprogram` 文件夹**（即包含 `app.json` 的那一层）。
3. AppID 选「**测试号**」（项目已预置 touristappid，可直接用测试号预览，无需注册）。
4. 进入后在模拟器即可查看；点工具栏「预览」生成二维码，用微信扫码可在真机体验。

> 提示：若导入时报「未找到 app.json」或「未找到 pages/index/index.wxml」，多半是只下载了单个文件导致目录被打散。请使用随附的 **zip 包**整体解压后再导入，目录结构会完整保留。

## 适配与说明

- 布局用 `rpx` 单位，自动适配不同尺寸手机；图表 canvas 按屏宽自适应，节点大小随杠杆缩放。
- 详情用底部弹出卡（bottom sheet），符合手机操作习惯；点遮罩或「关闭」收起。
- 画布文字使用系统字体（思源宋体未随包内置）。如需与网页版完全一致的宋体观感，可在 `app.js` 的 `onLaunch` 里用 `wx.loadFontFace` 加载 Noto Serif SC 的 woff2（需自备字体文件与合规授权），canvas 内文字则建议保留系统字体以保证性能。
- 打分口径与边界同网页版：三轴（抵押品强度 / 杠杆 / 反身性）为 1–5 序数判断，脆弱度 =（杠杆 + 反身性 +（6 − 抵押品））/ 3；它刻画结构脆弱度，是明斯基谱系的代理而非定义，最终归类取决于现金流能否覆盖本息。

## 基础库

`project.config.json` 设 `libVersion` 为 2.33.0；canvas 2d（`type="2d"`）需基础库 2.9.0 及以上。如遇旧版本，请在开发者工具「详情 → 本地设置」里调高调试基础库版本。
